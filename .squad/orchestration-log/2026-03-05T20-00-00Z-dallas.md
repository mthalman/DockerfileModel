## Technical Decisions (Implementation)
1. **Custom Parser monad** — `Parser α := Position -> ParseResult α`, providing `pure`, `bind`, `fail` with zero external dependencies.
2. **Token-producing** — All parsers return `Token` values, not strings, to preserve structure for round-trip proofs.
3. **Three-tier module organization** — Basic (monad), Combinators (patterns), DockerfileParsers (app-specific).
4. **`partial` keyword** — Used for `many`, `bracedVariableRef`, `literalWithVariablesUnquoted` where termination is guaranteed by progress but hard to prove structurally.
5. **Round-trip theorems with `sorry`** — Statements are precise; proofs deferred to Phase 3+.
